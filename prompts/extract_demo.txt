# Your Task
Extract business rules and operational details from the demo call transcript below and output a structured JSON document following the Account Memo schema.

# Critical Rules (READ CAREFULLY)

## 1. NEVER HALLUCINATE DATA
- If information is not explicitly stated in the transcript, DO NOT invent it
- Use `null` for missing fields
- Add missing information to the `questions_or_unknowns` array
- It is BETTER to flag something as unknown than to guess

## 2. DEMO CALLS ARE EXPLORATORY
- Demo calls provide directional information, not precise operational details
- Expect many unknowns - this is normal for v1
- Focus on capturing the general business model and requirements
- Specific details (exact phone numbers, precise hours) often come later in onboarding

## 3. ACCOUNT ID GENERATION
- Extract company name from transcript
- Generate account_id as lowercase slug: "company-name" → "company-name"
- Example: "Acme Fire Protection" → "acme-fire-protection"
- Example: "Bob's HVAC Services" → "bobs-hvac-services"

## 4. WHAT TO EXTRACT

### Business Hours
- Days of operation (Monday, Tuesday, etc.)
- Start time (HH:MM format, 24-hour)
- End time (HH:MM format, 24-hour)
- Timezone (America/New_York, America/Los_Angeles, etc.)
- If ANY component is unclear → set to null and add to unknowns

### Services Supported
- List all services mentioned (e.g., "fire protection", "sprinkler systems", "fire alarms")
- Use lowercase, descriptive names
- Include both emergency and non-emergency services

### Emergency Definition
- What situations count as emergencies?
- Examples: "sprinkler leak", "fire alarm triggered", "gas leak", "no heat in winter"
- Be specific - extract exact phrases used in the call

### Routing Rules
- Emergency contacts (primary, secondary)
- Non-emergency routing (business hours vs after hours)
- Call transfer rules (timeout, retries, fail message)
- Phone tree structure if mentioned

### Integration Constraints
- ServiceTrade rules (e.g., "never create sprinkler jobs in ServiceTrade")
- Other system limitations or requirements
- Data sync rules

### Flow Summaries
- Briefly describe the business hours call flow
- Briefly describe the after-hours call flow
- Keep these concise (2-3 sentences each)

## 6. QUESTIONS OR UNKNOWNS FORMAT

Use clear, actionable phrases:
- "missing: [what's missing]"
- "unclear: [what needs clarification]"
- "not discussed: [what wasn't covered]"

Examples:
- "missing: exact business hours (only mentioned 'regular hours')"
- "unclear: whether sprinkler emergencies go to on-call technician or manager"
- "not discussed: after-hours voicemail handling"

## 7. COMMON PATTERNS IN TRADE BUSINESSES

### Fire Protection / Sprinkler Companies
- Emergency: leaks, alarm triggers, system failures
- Services: inspection, installation, repair, monitoring
- Integration: ServiceTrade for work orders
- Routing: Often have on-call technicians 24/7

### HVAC Companies
- Emergency: no heat (winter), no AC (summer), gas smell
- Services: installation, maintenance, repair
- Seasonal hours variations common

### Electrical Services
- Emergency: power outage, sparking, burning smell
- Services: installation, troubleshooting, panel upgrades
- Safety-critical nature affects routing
